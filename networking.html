Day 1: OSI Model & Data Types

1

OSI Model Layers & Data Types

On the first day, we learned about the OSI (Open Systems Interconnection) model and different types of data.

7 Layers of OSI Model:

  1. Physical Layer - Transmission of raw bit streams over physical medium
  2. Data Link Layer - Error detection and correction, MAC addresses
  3. Network Layer - Logical addressing and routing (IP addresses)
  4. Transport Layer - End-to-end connections and reliability (TCP/UDP)
  5. Session Layer - Establishing, managing, and terminating connections
  6. Presentation Layer - Data translation, encryption, and compression
  7. Application Layer - Network services to applications (HTTP, FTP, SMTP)

Types of Data:

Voluntary Data

Information that users knowingly provide, such as registration forms, survey responses, or preferences.

Observed Data

Information collected by observing user actions, like website browsing patterns, purchase history, or location data.

Inferred Data

Information derived from analyzing voluntary and observed data to make predictions about preferences, behaviors, or characteristics.

Day 2: DHCP Concepts

2

Dynamic Host Configuration Protocol

On the second day, we explored DHCP (Dynamic Host Configuration Protocol) and its related concepts.

Key DHCP Concepts:

  • Automatically assigns IP addresses to devices on a network
  • Uses a client-server model with DORA process (Discover, Offer, Request, Acknowledge)
  • Manages IP address allocation through leases with renewal timers
  • Provides other network configuration parameters (subnet mask, default gateway, DNS servers)
  • Reduces manual configuration effort and prevents IP conflicts

DHCP Process:

# DHCP Transaction Process (DORA)
1. DHCPDISCOVER - Client broadcasts to find available DHCP servers
2. DHCPOFFER - Server responds with an IP address offer
3. DHCPREQUEST - Client requests the offered IP address
4. DHCPACK - Server acknowledges and assigns the IP address lease

DHCP Server

Responsible for managing IP address pools and assigning addresses to clients.

DHCP Client

Device that requests network configuration parameters from a DHCP server.

IP Lease

Temporary assignment of an IP address with a specific duration before renewal.

Day 3: HTTP & HTTPS

3

Web Protocols & Security

On the third day, we learned about HTTP, HTTPS, and related web protocols.

HTTP (HyperText Transfer Protocol):

  • Application layer protocol for transmitting hypermedia documents
  • Uses a request-response model between clients and servers
  • Stateless protocol (each request is independent)
  • Uses TCP port 80 by default
  • Request methods: GET, POST, PUT, DELETE, HEAD, OPTIONS, etc.

HTTPS (HTTP Secure):

  • Extension of HTTP with encryption for secure communication
  • Uses TLS/SSL to encrypt the data transferred
  • Uses TCP port 443 by default
  • Provides authentication, data integrity, and confidentiality
  • Indicated by padlock icon in browser address bar
# Example HTTP Request
GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html,application/xhtml+xml
Connection: close

SSL/TLS

Cryptographic protocols designed to provide communications security over a network.

Digital Certificates

Electronic documents used to prove the ownership of a public key, issued by Certificate Authorities.

Handshake Protocol

Process of establishing a secure connection between client and server before data transmission.

Day 6: Baatchit Application

6

Project Presentation

On the final day, we presented our completed application named "Baatchit" - a secure messaging platform with cybersecurity features.

Baatchit Features:

  • End-to-end encryption for all messages
  • Self-destructing messages with configurable timer
  • Two-factor authentication for enhanced security
  • Secure file sharing with encryption
  • Message recall functionality
  • Screen security to prevent screenshots
